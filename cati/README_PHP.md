# Универсальный PHP для автоматического определения домена

## Описание

Созданы универсальные PHP файлы, которые автоматически определяют текущий домен и используют его в шапке и футере сайта.

## Файлы

1. **config.php** - Конфигурационный файл с функциями для работы с доменом
2. **header.php** - Шапка сайта с автоматическим определением домена
3. **footer.php** - Футер сайта с доменом

## Функции в config.php

- `getDomain()` - Возвращает текущий домен (с www, если есть)
- `getSiteUrl()` - Возвращает полный URL сайта (с протоколом)
- `getDomainWithoutWww()` - Возвращает домен без www
- `displayDomain()` - Безопасный вывод домена для HTML (с экранированием)

## Использование

### Вариант 1: Использование header.php и footer.php

В начале каждого PHP файла:
```php
<?php require_once 'header.php'; ?>
```

В конце каждого PHP файла:
```php
<?php require_once 'footer.php'; ?>
```

### Вариант 2: Использование только функций из config.php

В любом PHP файле:
```php
<?php
require_once 'config.php';
$domain = displayDomain();
?>
```

Затем используйте `<?php echo $domain; ?>` где нужно.

## Пример использования в шапке

Домен будет автоматически отображаться в:
- Логотипе: "Anubis - example.com"
- Футере: "© 2024 Anubis - example.com"
- Email в политике конфиденциальности: "info@example.com"

## Миграция HTML в PHP

Чтобы использовать эти файлы:

1. Переименуйте `.html` файлы в `.php`
2. Замените статическую шапку на `<?php require_once 'header.php'; ?>`
3. Замените статический футер на `<?php require_once 'footer.php'; ?>`
4. Обновите ссылки в навигации (index.php вместо index.html)

## Безопасность

Все функции используют `htmlspecialchars()` для защиты от XSS атак при выводе домена в HTML.
